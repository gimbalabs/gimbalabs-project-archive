# Asteria Frontend with T3App framework and Mesh

## Gimbalabs Primary Driver
To provide open, replicable safe spaces to learn, to explore, and to empower individuals and organizations anywhere so that we can solve meaningful problems

## Proposal Driver
Gimbalabs students need more opportunities to work on substantial projects to demonstrate their skills, build confidence, and establish their reputation. By providing a safe, replicable learning environment, Gimbalabs enables students to engage with both UTXO blockchain principles and frontend development skills.

## Proposal

### 1. Title or Brief Description
Asteria App - Build a new frontend for the Asteria game with the T3 App framework and Mesh.

### 2. Background or Rationale
The major stakeholders of the project are Asteria project by TxPipe, Gimbalabs community of learners, Andamio network, and Mesh.

**Asteria:** A blockchain-based game, powered by a series of smart contracts and a Cardano bot challenge, designed to demonstrate the potential and versatility of the eUTxO model on the Preprod net.

**Gimbalabs learners:** At Gimbalabs, students engage in project-based learning (PBL) courses and practical projects to understand Cardano development and blockchain fundamentals.

**Andamio Network:** The Andamio network provides a CMS and contribution management platform.

**Mesh:** Provides a library to set up interactions with the blockchain.

The Asteria game fits naturally within this PBL framework as a student project that helps learners understand the UTXO model and Cardano frontend development.

### 3. Intended Outcome

**Goals:**
1. **Develop Scaffolding and Documentation**: Provide clear documentation of the development process to guide future students in completing the project.
   - Output: a list of resources, links, tool docs, references.
   - **Phase 0**
2. **Learn Cardano frontend development:** Rebuild the Asteria frontend using Mesh (instead of Lucid) and the T3 framework.
   - Output: a github repo with a working Asteria frontend that uses Mesh and T3
   - **Phase 1**
3. **Create a learning project template**: by combining resources from TxPipe, Mesh, and T3.
   - Output: a T3 based front end development course on Andamio
   - **Phase 2**
4. **Foster Collaboration**: Strengthen ties between the Gimbalabs and TxPipe communities through shared goals, resources, and mutual participation.
   - Output: TxPipe and GL contributions in the form of posts on discord and attendance at open meetings
   - **All Phases**
5. **Create a focused learning experience** with the Cardano web app tech stack: T3 app framework and components - Next.js, TypeScript, tRPC, Prisma, TailwindCSS, and NextAuth.js as well as Mesh and Aiken
   - Output: And, a dev log - learning journal (ie questions and answer for Advisor, work logs, etc.)
   - All Phases
6. **Be a bridge from learning to contribution**: Empower students to confidently explore and contribute to the Cardano app development and open-source projects.
   - Output: a resource for all the PBL students to Cardano learn frontend dev
   - All Phases

### 4. Details of the Proposal

Three Phases: First two are directly in scope. The last phase is something that will happen later, and requires another proposal.
1. Orientation: Research and familiarization with Asteria game and the frameworks.
2. The Asteria frontend project: Create (build and document) the building phase
3. The student project template: Convert our documentation into a student project template for Andamio Contribution management system.

#### Phase 0: Research and document
1. Collect tutorials, links to the documentation, planning, connecting with helpers (introduce ourselves on Discord and forum), play the Asteria game with a focus on rebuilding the frontend
2. Do a project management process in Notion - the money / compensation. See the Resource section
3. Help from Advisor in biweekly sessions
   - Break things down
   - Report and discuss progress

#### Phase 1: Building the Asteria front end with T3 and Mesh
1. Create a collaborative GitHub repo
   - Open development style - keeping learning journals / dev journal - document the journey for accountability and motivation
2. Follow tutorials and learning resources and update the Repo
3. Builders meet periodically to coordinate effort.
4. Presenting at the monthly proposal review at GL open spaces
5. Builders and advisor meet biweekly for mentor sessions
6. Ask questions in Cardano Developer / Tooling forums and Discord

#### Phase 2: The student project template (Secondary Scope)
Phase 2 is an indirect objective and in the secondary scope of the project. It is detailed here as an organizer for our "in scope" work. Basically, the first parts of the project build up to this and gives us the reason to slowly document the process, work as a team, and build with the door open. Hopefully, the outputs that we create will become the source material for phase 2.

1. Create a set of deliverables:
   - A list of the prerequisite skills needed for the project
   - A list of tasks and sub-tasks to complete the project
   - A set of Acceptance Criteria for each task
   - A simple document or playbook - tutorial that guides thru the building Asteria and playing it.
   - An introduction to the Asteria maintainers and team at TxPipe. "They know who I am and what I'm doing"... and they want to help (because I'm helping their KPI)
2. A project - a start, a middle, a challenge, and an end.
   - References to course materials and Student learning targets
   - Mini assignments or sub tasks - that create a roadmap or recipe for completing the project
   - End is a celebration and recognition of achievement
   - This is a PBL side quest. For example, it could branch from this section of the PPBL - https://www.andamio.io/course/ppbl2024/201/assignment/assignment201

   - Strengthen skills and reputation in app development
   - Increase skills and reputation in Cardano integration
   - Provide an avenue of meaningful contribution
